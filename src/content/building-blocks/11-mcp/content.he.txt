## הבעיה - כל חיבור דורש בנייה מחדש

בפרק הקודם ראינו איך API מאפשר לסוכן לדבר עם שירותים חיצוניים. אבל יש בעיה: כל חיבור חדש דורש בנייה מאפס. רוצים לחבר את הסוכן לגוגל דרייב? בונים חיבור ייעודי. רוצים לחבר ל-GitHub? עוד חיבור. כל פלטפורמה, כל שירות - עוד "צינור" שצריך לבנות מאפס.

תחשבו על מחשבים בשנות ה-80: כל מכשיר חיצוני (מדפסת, עכבר, מסך) דרש חיבור שונה וכבל שונה. ואז הגיע **USB** - חיבור אחד אחיד שמתאים לכולם.

**Model Context Protocol - MCP** הוא ה-USB של עולם ה-AI.


## מה זה MCP?

MCP הוא **פרוטוקול** - סט כללים אחיד שמאפשר לכל מודל AI להתחבר לכל מקור מידע או כלי חיצוני. במקום לבנות חיבור ייעודי לכל שירות, יש "שפה" אחת שכולם מדברים.

**הרעיון הפשוט:** במקום שכל מודל AI יצטרך ללמוד "לדבר" עם כל שירות בנפרד, MCP אומר: "כולם ידברו בפורמט אחיד, ומי שרוצה להתחבר - פשוט יבנה מתרגם קטן."

יש שלוש דמויות בכל חיבור MCP:

**1. Host (מארח)** - האפליקציה שאתם עובדים בה. למשל Claude Desktop או VS Code. היא מכילה את מודל ה-AI, מנהלת את החיבורים, ומחליטה למי מותר לדבר עם מי.
**2. Client (לקוח)** - ה"מחבר" שיושב בתוך ה-Host. לכל שרת MCP יש Client נפרד שמקשר בינו לבין ה-Host. כשהמודל צריך לקרוא קובץ או לחפש באינטרנט, ה-Host שולח את הבקשה דרך ה-Client המתאים.
**3. Server (שרת MCP)** - ה"מתרגם". קוד קטן שיושב בין ה-Host לבין המידע. הוא אומר: "אני יודע לקרוא קבצים, הנה מה שיש". כל שרת MCP מתמחה בדבר אחד - קבצים, GitHub, Google Drive, בסיסי נתונים.

## איך זה עובד בפועל?

**1.** אתם שואלים את Claude: "מה כתוב בקובץ notes.txt?"
**2.** Claude (בתוך ה-Host) מבקש לקרוא קובץ, וה-Host שולח את הבקשה דרך ה-Client לשרת ה-MCP המתאים
**3.** ה-Host בודק: "יש שרת MCP לקבצים? כן. יש לו הרשאה לתיקייה הזו? כן." - ומעביר את הבקשה
**4.** שרת ה-MCP קורא את הקובץ ומחזיר את התוכן
**5.** Claude מקבל את המידע ועונה לכם

כל התהליך הזה קורה אוטומטית - אתם פשוט שואלים שאלה.

## MCP לעומת אוטומציה מסורתית

יש כלים כמו **n8n** (שנכיר בפרק הבא) שמאפשרים לבנות **תהליכים אוטומטיים עם כמה שלבים** - למשל "כשמגיע מייל חדש, תסכם אותו, תשמור ב-Google Sheets ותשלח התראה בטלגרם". זה Workflow עם לוגיקה שאתם מגדירים מראש.

**MCP** שונה - כאן **ה-AI מחליט לבד** מתי ואיך להשתמש בכלים. אתם לא בונים Workflow - אתם נותנים לסוכן יכולות, והוא מפעיל אותן לפי הצורך.

בפועל, הרבה מערכות משתמשות בשתי הגישות ביחד.

## אבטחה - למה MCP בטוח

שלושה עקרונות חשובים:

**1. בידוד (Sandboxing)** - אתם מגדירים בדיוק לאילו תיקיות או שירותים מותר לשרת MCP לגשת. אם הגדרתם גישה רק לתיקייה מסוימת, השרת פיזית לא יכול לראות שום דבר אחר.
**2. ריצה מקומית** - שרת ה-MCP רץ על המחשב שלכם. המידע לא עובר לשרת חיצוני. הוא עובר ישירות מהקובץ שלכם ← לשרת המקומי ← ל-Claude.
**3. הפרדת מפתחות** - מפתחות API ו-Tokens נשמרים בהגדרות השרת על המחשב שלכם. ה-AI אף פעם לא רואה אותם - הוא רק מקבל את התוצאה הסופית.

## למה זה חשוב

MCP משנה את המשחק כי הוא הופך את ה-AI מ"צ'אטבוט חכם" ל-**ישות שמחוברת לעולם**. במקום לבנות חיבור ייעודי לכל שירות, יש סטנדרט אחד שמאפשר לכל AI להתחבר לכל דבר.

וזה בדיוק מה שמאפשר לכלים כמו Claude CLI או Cursor לעבוד כמו סוכנים אמיתיים - הם פשוט מתחברים לכל מה שצריך דרך MCP, בלי שאתם מרגישים.
