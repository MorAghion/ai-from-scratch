זה הנספח המלא - כל שלבי תרגיל הבלש ממוינים לפי סדר הפרקים. אם עקבתם אחרי התרגיל לאורך הספר, אתם כבר עשיתם את הכל. אם קפצתם ישר לכאן - זה המדריך המלא.

**מה בניתם:**

סוכן AI בלשי בסגנון בנואה בלאנק (Knives Out) שיודע:
- לקרוא זירת פשע ולנתח אותה
- לסרוק מספר קבצי ראיות ולמצוא סתירות
- לדרג חשודים לפי סבירות
- להתחבר לקבצים דרך MCP

---

**מפת השלבים:**

**פרק 2 - יצירת זירת הפשע**
בצ'אט חינמי (GPT/Gemini/Claude), יצרתם קובץ crime_scene.txt עם 4 חשודים, ארוחת ערב באחוזה, ורמזים עדינים. כל קורא יצר תיק שונה - זה החלק הכיפי.

**פרק 3 - הנדסת פרומפטים**
הפכתם את ה-AI לבלש עם System Prompt שכולל: Role (בנואה בלאנק), Chain of Thought (חשוב לפני שמסיק), ו-Ranking (דירוג 1-10). שלחתם לו את זירת הפשע וקיבלתם ניתוח ראשון.

**פרק 4 - מצ'אט לסוכן**
יצרתם את מבנה הפרויקט detective-ai/ עם crime_scene.txt בתוכו. עדיין עבדתם דרך צ'אט, אבל הכנתם את הקרקע לקוד.

**פרק 5 - בניית הסוכן**
התקנתם סביבת פיתוח (VS Code, Python, API Key) ובניתם את detective.py - סקריפט שקורא את crime_scene.txt ושולח ל-API אוטומטית. הקפיצה מצ'אט ידני לסוכן אוטומטי.

**פרק 6 - טוקנים וזיכרון**
בדקתם מה קורה כשהתיק גדל - זמן תשובה עולה, פרטים "נשכחים". גילו את מגבלות ה-Context Window בפרקטיקה.

**פרק 7 - מרחב וקטורי**
חקרתם Semantic Similarity - מצאתם ראיות שקשורות במשמעות אבל לא במילים. הבנתם למה חיפוש וקטורי עדיף על חיפוש מילולי.

**פרק 8 - RAG**
הוספתם תיקיית evidence/ עם עדויות חשודים ודוח נתיחה. שדרגתם את detective.py לקרוא מספר קבצים ולתייג כל מקור. הבלש התחיל למצוא סתירות בין עדויות.

**פרק 9 - אמינות**
בדקתם אם הבלש ממציא עובדות (הזיה). שדרגתם את ה-System Prompt עם הוראות Ground Truth - "ציטט מקורות, אל תמציא." ראיתם את ההבדל.

**פרק 10 - API**
הסתכלתם מתחת למכסה - ראיתם את ה-JSON המלא של ה-Request וה-Response. הבנתם שכל "מחשבה" של הבלש היא קריאת API.

**פרק 11 - MCP**
חיברתם את הבלש דרך MCP - במקום סקריפט Python שקורא קבצים, הסוכן ניגש לראיות ישירות. אותה תוצאה, אפס קוד.

**מה כל שלב לימד אתכם:**

| שלב | קונספט | מה הבלש למד לעשות |
|------|---------|-------------------|
| זירת פשע | Prompt Engineering | ליצור תוכן מובנה |
| System Prompt | Role + CoT + Constraints | לחשוב לפני שמסיק |

| טוקנים | Context Window | להבין את המגבלות שלו |
| ראיות מרובות | RAG | לשלב מידע ממקורות שונים |
| Ground Truth | Integrity | לציטט במקום להמציא |
| API | Request/Response | להבין מה קורה מתחת למכסה |
| MCP | Universal Protocol | לגשת לכלים בלי קוד מותאם |
