## מה זה אוטומציה בעולם AI?

בפרקים הקודמים למדנו איך סוכן AI קורא כלים דרך Function Calling, איך הוא שולף מידע דרך RAG, ואיך הוא מתקשר עם שירותים דרך API. עכשיו השאלה היא: **איך מחברים את כל זה ביחד למשהו שעובד לבד?**

זו בדיוק השאלה שכלי אוטומציה באים לענות עליה.

דמיינו מפעל קטן: חומר גלם נכנס מצד אחד, עובר תחנות עיבוד, ויוצא כמוצר מוגמר מהצד השני. אף אחד לא צריך ללחוץ על כפתור בין תחנה לתחנה - הכל זורם אוטומטית.

## n8n - בונים אוטומציה בלי קוד

**n8n** (נקרא "נוד-אייטי-אן") הוא כלי Low-Code שמאפשר לבנות תהליכים אוטומטיים בצורה ויזואלית. במקום לכתוב קוד שמחבר בין שירותים, גוררים "קוביות" על מסך ומחברים אותן בקווים.

**איך זה עובד?**

כל קובייה נקראת **Node** (צומת), וכל קובייה עושה דבר אחד: שולחת מייל, קוראת מ-Google Sheets, שואלת שאלה מ-LLM, או כל פעולה אחרת. מחברים אותן ב-**Workflow** - רצף צמתים שרץ אוטומטית.

**דוגמה פשוטה:** מישהו שולח הודעה בטלגרם ← n8n קולט את ההודעה ← שולח אותה ל-Gemini ← מקבל תשובה ← שולח חזרה לטלגרם. ארבעה צמתים, אפס שורות קוד.

**מה מתחיל את הכל?** כל Workflow מתחיל ב-**Trigger** - האירוע שמפעיל את התהליך. הודעת טלגרם, מייל שהגיע, זמן מסוים ביום, או קריאת Webhook מבחוץ.

**מה עובר בין הצמתים?** בכל "קו" שמחבר בין צמתים זורם **JSON** - אותו הפורמט שדיברנו עליו בפרק ה-API. כל צומת מקבל JSON, עושה את שלו, ומעביר JSON הלאה.

## Docker - הקופסה שמריצה את הכל

אם n8n הוא הכלי, **Docker** הוא הקופסה שהכלי רץ בתוכה.

**האנלוגיה:** דמיינו שקניתם משחק מחשב. בלי Docker, אתם צריכים להתקין את כל התוכנות הנלוות, לוודא שהגרסאות מתאימות, ולפתור בעיות תאימות. עם Docker - המשחק מגיע בקופסה סגורה שכוללת את הכל. פותחים, רץ.

**Container (מכולה):** קופסה מבודדת שמריצה תוכנה אחת. בתוכה יש את הקוד, הספריות, וכל מה שצריך. לא משנה אם אתם על מק, חלונות או שרת בענן - המכולה תעבוד אותו דבר.

**Image (תמונה):** ה"מתכון" שממנו נוצרת המכולה. קובץ סטטי שמכיל את כל ההוראות. אתם מורידים Image, ומפעילים ממנו Container.

**Volume (נפח):** זיכרון קבוע שנשמר גם כשהמכולה נכבית. בלי Volume, כל מה שהמכולה שומרת - נמחק כשהיא נסגרת. Volume זה כמו USB חיצוני שמחובר לקופסה.

## איך מרימים מערכת - בפועל

בפועל, משתמשים ב-**Docker Compose** - קובץ פשוט שמתאר אילו מכולות צריך להרים ואיך לחבר אותן.

**מה מגדירים בקובץ:**

**1.** אילו מכולות צריך (n8n, Vector DB, מודל AI)

**2.** Port Mapping - איזה "דלת" פתוחה בין המכולה לעולם החיצוני (למשל: פורט 5678 = גישה ל-n8n מהדפדפן)

**3.** Volume Mapping - חיבור תיקייה מהמחשב לתוך המכולה (כדי שה-AI יוכל לגשת לקבצים אמיתיים)

**4.** Environment Variables - משתני סביבה שמגדירים הרשאות והגדרות

פקודה אחת - `docker compose up` - ומערכת שלמה עולה לאוויר.

## חיבור סוכן AI לכלים דרך n8n

ברגע שהמערכת רצה, אפשר לבנות ב-n8n סוכן AI אמיתי:

**1.** מחברים **מוח** - Credentials של מודל שפה (Gemini, Claude, GPT)

**2.** מחברים **כלים** - צומת שקורא קבצים, צומת שמחפש באינטרנט, צומת ששולח מיילים

**3.** כותבים **System Prompt** - הנחיות שאומרות לסוכן "השתמש רק במידע מהכלים, אל תמציא"

**4.** מגדירים **Trigger** - מה מפעיל את הסוכן (הודעת טלגרם, Webhook, לוח זמנים)

זוכרים את ה-[Agentic Loop מפרק 4](#chapter:first-agent:הלולאה-הסוכנית---agentic-loop)? זה בדיוק מה שקורה פה. הסוכן מקבל שאלה, מחליט איזה כלי להפעיל, קורא את התוצאה, ומחליט אם לענות או לקרוא כלי נוסף.

## למה זה חשוב

n8n ו-Docker הם הגשר בין "אני מבין מה זה AI" לבין "יש לי מערכת AI שעובדת". הם מאפשרים לקחת את כל מה שלמדנו - LLM, סוכנים, Function Calling, RAG, API - ולחבר את הכל למשהו שרץ באמת.
