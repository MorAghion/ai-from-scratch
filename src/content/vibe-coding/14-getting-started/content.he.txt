## Git & GitHub

כשעובדים עם סוכן AI שכותב קוד, דבר אחד הוא קריטי: **גיבוי**. הסוכן עושה שינויים מהירים, לפעמים שוברים, ובלי היסטוריה של גרסאות - אין דרך חזרה.

**Git** הוא הכלי שעוקב אחרי כל שינוי בקוד שלכם. כל פעם שעושים *commit*, נשמרת תמונת מצב שאפשר לחזור אליה. **GitHub** הוא השירות שמאחסן את הקוד בענן - גיבוי, שיתוף, ונקודת התייחסות.

למה זה חשוב ב-Vibe Coding?
- הסוכן שבר משהו? חוזרים ל-commit הקודם
- רוצים לנסות כיוון אחר? פותחים branch
- עובדים על כמה מכונות? GitHub מסנכרן הכל

**הטיפ הכי חשוב:** עשו commit *לעיתים קרובות*. אחרי כל פיצ׳ר שעובד, אחרי כל תיקון משמעותי. לא לחכות לסוף היום. למדתי את זה the hard way כשהסוכן שכתב מחדש קובץ שלם ואיבדתי שעתיים עבודה.

## API Keys - מה זה ולמה צריך

במחברת הראשונה למדנו מה זה API - הדרך שבה מערכות מדברות זו עם זו. **API Key** הוא כמו סיסמה שמזהה אתכם מול השירות.

**מתי צריך API Key?**
- כשבונים אפליקציה שמשתמשת ב-AI (לא רק צ׳אט)
- כשעובדים עם Claude Code (הוא צריך גישה ל-API)
- כשמריצים סקריפטים שקוראים למודלים

**מתי לא צריך**?
- כשמשתמשים בצ׳אט דרך האתר (ChatGPT, Claude.ai)
- כשעובדים עם Cursor או Copilot (הם מנהלים את המפתחות בשבילכם)

**איך משיגים API Key**?
כל ספק API מציע דף שבו יוצרים מפתח. בדרך כלל תמצאו את זה תחת *Settings* → *API Keys*. יצירת המפתח עצמו היא בחינם - רוב השירותים (Google APIs, Supabase, Firebase ועוד) נותנים מפתח בלי לשלם שקל. רק ספקי מודלי AI כמו Anthropic ו-OpenAI דורשים אמצעי תשלום, כי אצלם התשלום הוא לפי שימוש בטוקנים.

**כמה עולה שימוש ב-API של מודלי AI**?
תלוי בשימוש. שיחת צ׳אט רגילה עולה סנטים בודדים. בניית פרויקט שלם יכולה לעלות $10-50 ליום אם לא נזהרים (כן, למדתי את זה על בשרי).

## ניהול Secrets

**Secret** הוא כל מידע שלא רוצים שייחשף - סיסמאות, טוקנים, ובעיקר API Keys. מי שמחזיק במפתח יכול לפעול בשמכם.
לפעמים תיתקלו גם ב-**Client Secret** - זה מפתח סודי שהשירות נותן לאפליקציה שלכם, בזוג עם **Client ID** גלוי. השניים ביחד מוכיחים שהאפליקציה היא באמת שלכם.

יש לכם API Key - מעולה. עכשיו **אל תשימו אותו בקוד.**

ברצינות. זה אחד הדברים הכי חשובים שצריך להפנים. אם המפתח שלכם מגיע ל-GitHub - מישהו ישתמש בו תוך דקות, ואתם תקבלו חשבון של מאות דולרים.

**הפתרון: קובץ .env**

קובץ `.env` הוא קובץ פשוט שיושב בתיקיית הפרויקט ומכיל את המפתחות שלכם:

```
ANTHROPIC_API_KEY=sk-ant-...
OPENAI_API_KEY=sk-...
```

הקוד שלכם קורא את המפתחות מהקובץ הזה, אבל הקובץ עצמו **לעולם לא עולה ל-Git**.

**איך מוודאים שזה לא עולה?** מוסיפים את `.env` לקובץ `.gitignore`:

```
.env
.env.local
```

**ומה קורה אם בטעות חשפתם מפתח?**
1. נכנסים מיד לדף ה-API של הספק
2. מוחקים (revoking) את המפתח הישן
3. יוצרים מפתח חדש
4. נושמים. למדתם משהו.
