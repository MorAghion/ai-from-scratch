## תכנון לפני ביצוע

אז מה בעצם הכוונה בתכנון?
בסוף, הסוכן הוא מי שבאופן מעשי כותב את הקוד של האפליקציה שלנו. לכן, השאיפה היא להגיע למצב שבו התיאור של האפליקציה, הדרישות, הפיצ׳רים והמערכת מתוארים בצורה הכי טובה שניתן - **לפני** שהסוכן מתחיל לעבוד. למה? כי **פחות תכנון = יותר יד חופשית לסוכן = פחות בקרה שלנו על הנעשה**. וזה משהו שאנחנו רוצים לצמצם. 
אז איך עושים את זה?

__**PRD - מסמך דרישות מוצר**__
**מה זה PRD?** מסמך דרישות מוצר (Product Requirements Document) — המסמך שמגדיר מה המוצר צריך לעשות לפני ששורה אחת של קוד נכתבת. חזון, פיצ׳רים, מסכים, אינטראקציות. לא איך לבנות — רק מה צריך לצאת בסוף.

**למה זה קריטי ב-Vibe Coding?** כי הסוכן צריך הקשר. בלי PRD, אתם מתארים כל פיצ׳ר בזמן אמת ומקווים שהסוכן יבין את התמונה הגדולה. עם PRD, הסוכן רואה את כל הפאזל מהרגע הראשון.

**ב-PRD שלי:**
- הגדרתי את ההיררכיה של: HomeHub: Hub → Sub-Hub → Master List / Active List. 
- לכל פיצ׳ר תיארתי לא רק מה הוא עושה - אלא גם את הפרטים הקטנים:
-- מה קורה כשמוסיפים פריט? 
-- מה קורה כשהרשימה ריקה?
-- מה קורה כששני משתמשים עורכים בו-זמנית? 
- הגדרתי שפה ויזואלית וחזון: "להפחית עומס קוגניטיבי. עיצוב רגוע, אוטומציה חכמה."

__**מסמך ארכיטקטורה**__
**מה זה מסמך ארכיטקטורה?** מגדיר איך המוצר נבנה מבחינה טכנית. אילו טכנולוגיות, למה בחרנו בהן, ואיך הן מתחברות זו לזו. לא מה האפליקציה עושה - אלא על מה היא עומדת ואיך השלד שלה נראה.

**למה זה קריטי ב-Vibe Coding?** כי הסוכן מקבל החלטות טכניות כל הזמן, ובלי הנחיה ברורה, כל סוכן יחליט לפי שיקול דעתו. התוצאה: פרויקט לא עקבי, עם ספריות שמתנגשות ובחירות שסותרות אחת את השנייה. עם מסמך ארכיטקטורה, כולם עובדים לפי אותם חוקים, והמערכת תהיה יציבה יותר.

**הארכיטקטורה שלי.** הגדרתי את כל הסטאק עם הנמקה: React 19, TypeScript, Vite, Tailwind, Supabase, Vercel. לכל בחירה כתבתי למה - ולמה לא האלטרנטיבה. הסוכן לא החליט לבד במה להשתמש, הוא פשוט ביצע.




## Claude Code - הכלי שלמדתי להשתמש בו

**Claude Code** הוא סוכן AI שרץ בטרמינל. הוא לא רק צ׳אט - הוא קורא קבצים מהפרויקט, כותב קוד, מריץ פקודות, רואה שגיאות ומתקן אותן. כל זה *לבד*, בלי שאתם עוברים ידנית בין טאבים.

מה שהבנתי: הכלי חזק כמו ההוראות שאתם נותנים לו. ובהתחלה לא נתתי כלום.

**CLAUDE.md - ספר ההוראות.** זה קובץ שיושב בשורש הפרויקט ו-Claude Code קורא אותו אוטומטית. שם כותבים:
- חוקי פרויקט (איזה framework, איזה סטנדרטים)
- מה מותר ומה אסור (permissions)
- איך עובדים עם Git (branches, PRs, commit messages)
- פרוטוקול לדיווח באגים

ה-CLAUDE.md שלי התחיל ריק. ב-22 בפברואר הוא היה 12,500 מילים. כל פעם שהסוכן עשה משהו שלא רציתי - הוספתי כלל ל-CLAUDE.md.

**תכנון ב-Claude Desktop, ביצוע ב-Claude Code.** זה היה ה-game changer. השתמשתי ב-Claude Desktop (הצ׳אט) לתכנון: *"מה הארכיטקטורה? איך מחלקים את העבודה? מה העדיפויות?"* ואז העברתי את התוכנית ל-Claude Code שביצע אותה. שני כלים, שתי מטרות.

**הסוכנים:** כל סוכן עם תפקיד ברור וצבע מזהה:
- **Coordinator** - תכנון, חלוקת משימות, עדכון הלוח
- **Architect** - סכמת בסיס נתונים, מיגרציות, החלטות טכניות
- **Frontend** - קומפוננטות, UI, תרגומים, תיקוני באגים
- **Backend** - Edge Functions, אבטחה, PWA
- **QA** - טסטים (unit, E2E), CI/CD, רגרסיה

**מערכת Handoffs:** כשסוכן מסיים משימה, הוא יוצר קובץ handoff - מסמך שאומר מה הושלם, מה הדליברבלס, ומה צריך לבדוק. הסוכן הבא קורא את ה-handoff ויודע בדיוק מאיפה להמשיך.

**דוגמה אמיתית:** ב-Wave 8 (25 בפברואר) הרצתי 6 משימות במקביל - 3 תיקוני באגים ו-3 חבילות טסטים. כולם על branches נפרדים, כולם עם PRs, כולם מורג׳ו באותו יום. **58 קומיטים ביום אחד**, והכל מסודר.

## התוצאה - HomeHub Live

מה נבנה בסך הכל?

- **4 הבים** מלאים: קניות, משימות, אוצ׳רים, הזמנות
- **Auth** מלא: אימייל + Google OAuth, שכחתי סיסמה, הצטרפות למשק בית
- **דו-לשוני**: עברית ואנגלית עם תמיכה מלאה ב-RTL
- **PWA**: עובד אופליין, ניתן להתקנה כאפליקציה
- **400+ טסטים**: unit, integration, E2E
- **CI/CD**: GitHub Actions, Vercel auto-deploy

המספרים: **226 קומיטים**, **40+ PRs**, **10 גלים**, **28 באגים** שזוהו ותוקנו.

**השוואת עלויות:**
- *Just Vibing* (8 ימים): ~245 ש״ח טוקני API + $100 מנוי Max = ~$167
- *שלב מובנה* (10 ימים): כלול במנוי Max - יותר פיצ׳רים, פחות עלות ליחידה

## ה-Backpropagation שלי

בדיוק כמו שרשתות נוירוניות לומדות מהשגיאות שלהן - גם אני למדתי מהשלי.

**היחס הכנה:** ה-AI כתב ~90% מהקוד. אני קיבלתי ~90% מההחלטות. *מה* לבנות, *איך* לחלק, *מתי* לעצור - זה היה עליי. הקוד עצמו? הסוכן.

**מה הייתי עושה אחרת?**
- מתחילה עם PRD ביום הראשון, לא ביום השמיני
- כותבת CLAUDE.md *לפני* שהסוכן עושה טעויות
- עושה commit כל 30 דקות, לא כל יום
- מייצבת פיצ׳ר אחד לפני שמתחילה את הבא

**שלושת הלקחים:**
1. **תכננו לפני שאתם מבצעים.** שעה של תכנון חוסכת יום של תיקונים.
2. **הסוכן טוב כמו ההוראות שלו.** CLAUDE.md זה לא אופציונלי - זו ההשקעה הכי משתלמת בפרויקט.
3. **טסטים הם לא אופציונליים.** הם מה שמאפשר לכם לזוז מהר בלי לשבור דברים.
