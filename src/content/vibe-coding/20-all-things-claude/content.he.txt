## Claude Code - צלילה עמוקה

זה הכלי שסביבו בנוי כל המסע הזה. אבל הרבה מהפיצ'רים שלו נשארים מוסתרים.

### CLAUDE.md - הזיכרון הקבוע של הפרויקט

קובץ markdown שיושב בשורש הפרויקט, ו-Claude Code קורא אותו אוטומטית בתחילת כל session. זה המקום שבו אתם מגדירים:
- חוקי הפרויקט (איך לעשות commits, מה אסור לגעת בו)
- ה-stack הטכני והארכיטקטורה
- prefernces אישיות (שפת תגובות, רמת verbose)
- פרוטוקולים חוזרים (איך לדווח באגים, איך לפתוח PR)

הכלי `/init` מייצר CLAUDE.md ראשוני אוטומטית על ידי סריקת הפרויקט.

אצלי ה-CLAUDE.md התחיל ריק. אחרי 10 ימים - 12,500 מילים. כל פעם שClaude עשה משהו שלא רציתי, הוספתי כלל.

[תיעוד CLAUDE.md](https://docs.anthropic.com/en/docs/claude-code/memory)

### Skills - פקודות מותאמות אישית

Skills הן slash commands שאתם מגדירים בעצמכם, כקבצי markdown בתיקיית `.claude/commands/`. שם הקובץ = שם הפקודה.

**איך מגדירים skill:**
- צרו קובץ `.claude/commands/commit.md`
- כתבו בו הוראות בפורמט markdown רגיל
- קראו לו בשיחה עם `/commit`

**דוגמה - skill לכתיבת commit messages:**

```
Write a git commit message for the staged changes.
Use conventional commits format: type(scope): description
Types: feat, fix, docs, refactor, test, chore
Keep subject line under 72 chars.
Add Co-Authored-By: Claude at the end.
```

Skills יכולות לקבל ארגומנטים דינמיים עם `$ARGUMENTS`, להפנות לקבצים אחרים, ולהיות משותפות לכל הצוות דרך ה-repo.

Claude יכול גם להחליט בעצמו מתי להפעיל skill - אם ביקשתם "תעשה commit מסוגנן", הוא יידע לרוץ על ה-commit skill שלכם.

[תיעוד Skills](https://docs.anthropic.com/en/docs/claude-code/tutorials#create-custom-slash-commands)

### Hooks - אוטומציה על גבי האוטומציה

Hooks הם פקודות shell שרצות אוטומטית לפני או אחרי כל פעולה של Claude Code. מוגדרים ב-CLAUDE.md או בהגדרות.

**PreToolUse** - רץ לפני שClaude מבצע פעולה. יכול לאשר, לדחות, או לשנות.
**PostToolUse** - רץ אחרי כל עריכה. שימושי ל: auto-format, type checking, הרצת טסטים.

דוגמה לשימוש: כל פעם שClaude עורך קובץ TypeScript - `tsc --noEmit` רץ אוטומטית. אם יש שגיאה, Claude רואה אותה ומתקן לפני שממשיך.

זה מה שהופך את Claude Code מכלי לinfrastructure.

[תיעוד Hooks](https://docs.anthropic.com/en/docs/claude-code/hooks)

## פיצ'רים מתקדמים ב-Claude Code

### Tool Search - כשיש יותר מדי כלים

כשמחברים הרבה MCP servers (GitHub + Slack + Postgres + Filesystem...) הגדרות הכלים לבדן אוכלות ~55K tokens עוד לפני שהתחלתם לעבוד. Tool Search פותר את זה.

במקום לטעון את כל הכלים לכל שאילתה, Tool Search טוען דינמית רק את 3-5 הכלים הרלוונטיים לכל בקשה ספציפית. הדיוק עלה מ-49% ל-74% בבדיקות של Anthropic.

הבשורה: **זה כבר פועל אצלכם ברירת מחדל** - Claude Code מפעיל Tool Search אוטומטית. אתם כבר נהנים ממנו בלי לדעת.

[תיעוד Tool Search](https://docs.anthropic.com/en/docs/agents-and-tools/tool-use/tool-search-tool)

### Agent Teams - הצוות המובנה

זכרו את הרעיון מפרק 17 - כמה סוכנים במקביל, כל אחד על המשימה שלו? Anthropic בנו את זה ישירות לתוך Claude Code.

**Agent Teams** מאפשרות לספawn כמה instances של Claude Code, כל אחד עם context window משלו, שמתקשרים ישירות אחד עם השני ועובדים על task list משותף. אחד מהם הוא Team Lead שמתאם.

**הפעלה:**
```
export CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS=1
claude
```

הכי שימושי ל: code review מקביל (security + performance + tests בו-זמנית), בניית מודולים עצמאיים, debugging עם השערות מתחרות.

עדיין experimental. אבל כבר עובד.

[תיעוד Agent Teams](https://code.claude.com/docs/en/agent-teams)

### Chrome Integration - Claude פותח דפדפן

דרך extension בשם "Claude in Chrome" ופקודת `claude --chrome`, Claude מקבל גישה חיה ל:
- ה-DOM של הדף
- console logs ו-stack traces
- network requests
- יכולת לקלিק, להקליד ולמלא טפסים - על דפים שאתם כבר מחוברים אליהם

זה לא MCP - זה Chrome DevTools Protocol ישירות.

**דוגמאות לשימוש של vibe coder:**
- "פתח את localhost:3000 ובדוק שהטופס מציג שגיאות נכון"
- "עבור על דף המוצרים ושלוף את כל המחירים ל-CSV"
- "בנה את ה-UI לפי העיצוב ואז פתח בדפדפן כדי לוודא שזה תואם"

Beta מאז מרץ 2026. עובד על Chrome ו-Edge בלבד (לא Brave, לא Arc).

[תיעוד Chrome Integration](https://code.claude.com/docs/en/chrome)


## יכולות מפתח של Claude

**חלון הקשר של 1M tokens** - זמין ב-Opus 4.6 ו-Sonnet 4.6. מאפשר לטעון קודבייס שלם, מסמך משפטי ארוך, או שנה של logs בשאילתה אחת. שימו לב: context גדול מאוד עלול להוריד דיוק בקצוות - הנקודה המתוקה היא עד ~200K לעבודה פרציזית.

**Vision - Claude רואה תמונות** - הדביקו screenshot של UI, שגיאה, או wireframe ישירות לשיחה. Claude מבין מה הוא רואה ומגיב בהתאם. עובד ב-Claude.ai, Claude Desktop, ודרך ה-API.

**Adaptive Thinking** - Claude מחליט בעצמו כמה לחשוב על כל בקשה. משימות מורכבות מקבלות chain-of-thought מלא, שאלות פשוטות מקבלות תשובה ישירה. זה המודל החדש - לא "extended thinking" שמפעילים ידנית. זמין ב-Opus 4.6 ו-Sonnet 4.6.

[תיעוד Adaptive Thinking](https://docs.anthropic.com/en/docs/build-with-claude/extended-thinking)

**Computer Use (Beta)** - Claude שולט במחשב ממש - פותח דפדפן, לוחץ על כפתורים, ממלא טפסים, מנווט בממשקים גרפיים. שימושי לאוטומציה של משימות שאין להן API. עדיין experimental - לא כדאי לסמוך עליו לפרודקשן.

[תיעוד Computer Use](https://docs.anthropic.com/en/docs/build-with-claude/computer-use)

**זיכרון מתמשך** - מאז מרץ 2026, Claude זוכר מידע עליכם בין שיחות - שם, סגנון תקשורת, העדפות, הקשר פרויקטים. זמין לכולם כולל Free tier. אפשר גם לייבא זיכרון מ-ChatGPT או Gemini. שווה לבדוק מה הוא שומר עליכם ולנהל את זה.

**Web Search** - Claude יכול לחפש באינטרנט לקבלת מידע עדכני. זמין ב-Claude.ai וב-Claude Desktop. בטרמינל (Claude Code) - לא זמין ברירת מחדל, אבל אפשר דרך MCP server של Fetch.

**Projects** - סביבות עבודה עם זיכרון ייעודי ב-Claude.ai. כל Project מקבל: context window של 200K לקבצי הפרויקט, הוראות מותאמות, וזיכרון שנשמר בין sessions. שימושי לפרויקטים עם הרבה הקשר קבוע.

**PDF Support** - Claude קורא PDF ישירות דרך ה-API (עד 100 עמודים, כולל טקסט, תמונות וטבלאות). שימושי לבניית כלים לעיבוד מסמכים.

**Google Workspace** - חיבורים רשמיים ל-Gmail, Google Drive, Google Calendar ו-Google Sheets. לא רק MCP - connectors מובנים שזמינים ב-Claude Desktop ו-Claude.ai.


## MCP - פרוטוקול החיבורים

### מה זה MCP בעצם

תחשבו על MCP כמו שקעי USB עבור AI. לפני MCP, כל חיבור בין Claude לכלי חיצוני דרש אינטגרציה מותאמת. MCP הוא התקן שמאפשר לכל כלי להתחבר לכל מודל AI עם אותו interface.

**לפני MCP:** אתם מדביקים נתונים לשיחה ידנית.
**עם MCP:** Claude מתחבר ישירות למקור ומושך מה שצריך.

MCP הוא open standard - אומץ גם על ידי OpenAI ו-Google DeepMind.

[תיעוד MCP](https://docs.anthropic.com/en/docs/mcp)

### שרתי MCP שכדאי להכיר

@@component:MCPServersTable

[ספריית שרתי MCP](https://github.com/modelcontextprotocol/servers)

### איך מוסיפים MCP ל-Claude Code

פתחו או צרו את הקובץ `~/.claude.json` והוסיפו:

```json
{
  "mcpServers": {
    "filesystem": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-filesystem", "/path/to/dir"]
    },
    "github": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-github"],
      "env": { "GITHUB_TOKEN": "your-token" }
    }
  }
}
```

אחרי הוספה, הפעילו מחדש את Claude Code. הקלידו `/help` לראות שהשרת הופיע. זה באמת יותר קל ממה שנשמע.
